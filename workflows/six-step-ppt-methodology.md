# PPT六步法 — 六角色协同工作流

> **定位**：这是 ppt-master 的"内容增强层"，在现有管线基础上叠加六步法，提升叙事连贯性和内容质量。
> **触发**：用户说"帮我做个PPT"、"做演示文稿"、或任何PPT生成需求时，在 Step 4 战略家阶段后自动激活。

## 六步法概述

| 步骤 | 角色 | 核心任务 | 输出 |
|------|------|----------|------|
| 1 | 演示文稿顾问 | 定义目标、受众、核心信息、幻灯片流程 | `content_blueprint.md` |
| 2 | 结构设计师 | 逐张幻灯片结构、标题、目的说明 | `slide_structure.md` |
| 3 | 故事叙述师 | 开场-问题-洞察-解决方案-结论 | `story_arc.md` |
| 4 | 视觉顾问 | 每页视觉指南、布局、图形、图表 | `visual_guidelines.md` |
| 5 | 内容生成器 | 每页完整内容、简洁专业要点 | `page_content.md` |
| 6 | 简化编辑器 | 审查、重写、减少文字、突出要点 | `final_content.md` |

## 第一步：演示文稿顾问 (The Consultant)

> **任务**：为[主题]创建蓝图，定义目标、受众、核心信息和幻灯片流程。

**输入**：
- 用户提供的主题/源材料
- 受众描述（如果没有，默认"行业专业人士"）

**输出格式**：
```markdown
# 演示文稿蓝图

## 基本信息
- 主题：[主题]
- 目标：[具体目标]
- 受众：[详细描述]
- 核心信息：[一句话概括]
- 预计页数：[数字]
- 时长：[分钟]

## 幻灯片流程
1. 封面页
2. 目录页
3. 背景/引言
4. 问题陈述
5. 洞察分析
6. 解决方案
7. 案例研究
8. 实施计划
9. 收益预测
10. 结论与展望
11. Q&A页
```

## 第二步：结构设计师 (The Structure Designer)

> **任务**：设计逐张幻灯片结构，确保每页有清晰标题和目的说明，流程自然流畅。

**输入**：
- `content_blueprint.md`（第一步输出）
- 源材料内容

**输出格式**：
```markdown
# 幻灯片结构设计

## 第1页：封面
- 标题：[主标题]
- 副标题：[副标题]
- 目的：建立第一印象，明确主题

## 第2页：目录
- 标题：议程概览
- 目的：引导观众，建立预期

## 第3页：引言
- 标题：[具体标题]
- 目的：[具体目的]
- 关键要点：
  - [要点1]
  - [要点2]
```

## 第三步：故事叙述师 (The Storyteller)

> **任务**：用"开场-问题-洞察-解决方案-结论"结构转化[主题]，保持专业与吸引力。

**输入**：
- `slide_structure.md`（第二步输出）
- 源材料

**输出格式**：
```markdown
# 故事弧线设计

## 开场 (Hook)
[用引人注目的事实/问题/场景吸引注意力]

## 问题 (Problem)
[描述当前面临的挑战/痛点]

## 洞察 (Insight)
[揭示关键发现/趋势/机会]

## 解决方案 (Solution)
[提出具体可行的方案]

## 结论 (Conclusion)
[总结要点，呼吁行动]

## 逐页故事流
| 页码 | 故事角色 | 情绪曲线 | 关键转折 |
|------|----------|----------|----------|
| 1-3 | 开场 | 好奇 | 引入主题 |
| 4-6 | 问题 | 紧张 | 痛点放大 |
| 7-9 | 洞察 | 启发 | 转折点 |
| 10-12 | 解决方案 | 信心 | 方案呈现 |
| 13-15 | 结论 | 鼓舞 | 行动号召 |
```

## 第四步：视觉顾问 (The Visual Advisor)

> **任务**：为每页建议视觉指南，推荐布局、图形、图表等提升清晰度和现代感。

**输入**：
- `slide_structure.md`（第二步输出）
- `story_arc.md`（第三步输出）

**输出格式**：
```markdown
# 视觉指南

## 整体视觉策略
- 配色方案：[推荐配色]
- 字体搭配：[主字体+辅助字体]
- 图标风格：[线性/面性/扁平]
- 图片风格：[摄影/插画/3D]

## 逐页视觉建议
### 第1页：封面
- 布局：全屏背景图+居中文字
- 图形元素：公司Logo、装饰线条
- 图表：无
- 视觉效果：大气、专业

### 第2页：目录
- 布局：左右分栏
- 图形元素：编号图标、分隔线
- 图表：无
- 视觉效果：清晰、有序
```

## 第五步：内容生成器 (The Content Generator)

> **任务**：创建每页完整内容，写简洁专业要点。

**输入**：
- `slide_structure.md`（第二步输出）
- `visual_guidelines.md`（第四步输出）
- 源材料

**输出格式**：
```markdown
# 页面内容

## 第1页：封面
- 主标题：[标题]
- 副标题：[副标题]
- 演讲者：[姓名/职位]
- 日期：[日期]

## 第2页：目录
- [章节1]
- [章节2]
- [章节3]

## 第3页：引言
- 标题：[标题]
- 正文：[具体内容]
- 关键数据：
  - [数据1]
  - [数据2]
```

## 第六步：简化编辑器 (The Simplified Editor)

> **任务**：审查内容重写适配幻灯片，减少文字突出要点，确保每页一个核心观点。

**输入**：
- `page_content.md`（第五步输出）

**审查清单**：
- [ ] 每页是否只有一个核心观点？
- [ ] 文字是否足够简洁？（每页不超过6行，每行不超过15字）
- [ ] 是否突出了关键数据？
- [ ] 语言是否专业且易于理解？
- [ ] 是否有冗余信息需要删除？

**输出格式**：
```markdown
# 最终内容

## 第1页：封面
✅ 通过
- 主标题：[标题]
- 副标题：[副标题]

## 第2页：目录
✅ 通过
- [章节1]
- [章节2]
- [章节3]

## 第3页：引言
⚠️ 需要简化
- 原标题：[原标题]
- 简化后：[简化标题]
- 删除原因：[原因]
```

## 与现有ppt-master技能的整合

### 整合位置
六步法在现有管线的 **Step 4（战略家阶段）** 之后、**Step 5（图像获取）** 之前激活。

### 整合方式
1. **Step 4 输出增强**：在生成 `design_spec.md` 和 `spec_lock.md` 的同时，生成 `content_blueprint.md`
2. **Step 5 前置**：在图像获取前，先完成六步法的内容生成
3. **Step 6 输入增强**：执行器阶段使用六步法的最终内容作为SVG生成的输入

### 可选激活
六步法不是强制的，用户可以选择：
- **完整模式**：运行全部六步
- **快速模式**：只运行第1、2、5步（顾问+结构+内容）
- **自定义模式**：指定需要运行的步骤

## 使用示例

### 示例1：完整模式
```
用户：帮我做一个关于人工智能在医疗领域应用的PPT
AI：好的，我将使用PPT六步法为您创建专业的演示文稿。
    1. 演示文稿顾问：定义目标、受众、核心信息
    2. 结构设计师：设计逐张幻灯片结构
    3. 故事叙述师：用故事弧线串联内容
    4. 视觉顾问：为每页建议视觉指南
    5. 内容生成器：创建每页完整内容
    6. 简化编辑器：审查并简化内容
    完成后将进入图像获取和执行器阶段。
```

### 示例2：快速模式
```
用户：快速做个PPT，主题是数字化转型
AI：好的，我将使用快速模式：
    1. 演示文稿顾问
    2. 结构设计师
    5. 内容生成器
    完成后直接进入图像获取阶段。
```

## 质量标准

### 内容质量
- 每页一个核心观点
- 文字简洁有力
- 数据支撑论点
- 逻辑连贯流畅

### 叙事质量
- 开场引人入胜
- 问题描述清晰
- 洞察深刻独到
- 解决方案可行
- 结论鼓舞人心

### 视觉质量
- 布局合理
- 配色协调
- 图标一致
- 图表清晰

## 注意事项

1. **不要过度设计**：六步法是指导原则，不是硬性规定
2. **保持灵活性**：根据用户需求和源材料调整步骤
3. **注重实用性**：最终目标是生成高质量的PPT，不是完美主义
4. **尊重用户偏好**：如果用户有特殊要求，优先满足用户需求
